# Progress Directory Guide

## Purpose

`docs/progress/` 存放阶段进展文档。

这里的文档回答的是：

- 哪些目标或方案已经推进到什么状态
- 哪些问题已经暴露
- 哪些阻塞、风险、偏差需要被跟踪
- 哪些结论已经被实施工作验证

## Rules

- 这里记录事实和状态，不承担长期目标定义职能。
- 不要把新的架构边界直接先写进 `progress/`；如果是长期约束，应先更新 `docs/targets/`。
- 不要把未来落地方案长期存放在这里；如果是实施设计，应写进 `docs/plans/`。
- 进度文档可以引用 `ST-xx` 和对应方案文档，但不应替代它们。
